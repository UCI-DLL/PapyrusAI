# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately.

- Do NOT open a public GitHub issue
- Do NOT disclose the issue publicly

Instead, please contact:
ucipapyrusai@gmail.com

We will review and respond as quickly as possible.

---

## What to Report

Examples of security issues include:
- Exposed API keys or credentials
- Authentication or authorization issues
- Data leaks or unintended access
- Vulnerabilities in dependencies

---

## Notes

- This repository contains only the frontend application
- Backend and infrastructure are managed separately
- Sensitive configuration is handled via AWS Amplify environment variables
